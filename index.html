Design Do's & Don'ts

Follow these best practices to maintain design excellence and brand consistency

âœ“ Do
  • Use Lato font family consistently across all text
  • Maintain 30px border-radius for all buttons
  • Apply 10px border-radius to cards and containers
  • Use #008CB6 for primary actions and links
  • Keep button minimum touch target at 44px on mobile
  • Use Title Case for headings and buttons
  • Apply proper spacing: 20-35px mobile, 40-60px desktop
  • Maintain consistent image heights with object-fit: cover
âœ— Don't
  • Never use border-radius below 25px for buttons
  • Avoid using multiple font families in one design
  • Don't make buttons full-width on desktop
  • Never use lowercase for button text
  • Avoid inconsistent color codes (use uppercase hex)
  • Don't use border-radius other than 6px, 10px, or 30px
  • Never skip hover states on interactive elements
  • Avoid cramped spacing or excessive whitespace

Accessibility Standards

WCAG 2.1 AA compliant design ensuring inclusive experiences for all users

Color Contrast

  • Primary Blue on white: 4.5:1 ratio
  • Black text on white: 21:1 ratio
  • Yellow with black text only
  • Never use yellow for body text

Touch Targets

  • Minimum 44px Ã— 44px size
  • Adequate spacing between targets
  • Full-width buttons on mobile
  • Avoid clustered clickable elements

Typography

  • Minimum 16px for body text
  • Line-height of 1.5+ for readability
  • Avoid long paragraphs (max 75 chars)
  • Use headings for proper structure

Focus States

  • Visible focus indicators always
  • Blue outline for keyboard navigation
  • Never remove outline without replacement
  • Test with keyboard navigation
